Observations

  • Inline Editing Mask: When in Edit Mode, an inline editing mask (<div class="dnnInlineEditingMask">) can sometimes intercept pointer events. Clicking elsewhere on the page dismisses this mask and allows normal module interaction.
  • Module Toolbars: Module action toolbars (Admin, Move) only appear when hovering directly over the module container. The toolbars provide quick access to management functions.
  • Pane Naming Convention: Content Layout panes use a prefix-based naming system (e.g., F30A_Pane1, F30A_Pane2). This prefix is generated by the LayoutPanePrefixGenerator and stored in the database.
  • Move Menu Pane List: The Move dropdown shows both skin panes (ContentPane, LeftPane, RightPane) and Content Layout panes. This provides flexibility in module placement across different pane types.
  • Empty Pane Placeholder: Empty panes display the CSS class DNNEmptyPane EditBarEmptyPane and show three placeholder icons for adding text, images, or modules.
  • Code Reference: The ProcessModules() method in View.ascx.cs (line 359-386) handles the placement of modules based on their PaneName property, moving module controls to the correct LayoutPane container.
