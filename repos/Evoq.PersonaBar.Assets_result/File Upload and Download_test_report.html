Observations

  • Inline vs Attachment Download: Based on code review, the Download API supports both inline (forceDownload=false) and attachment (forceDownload=true) modes. The UI defaults to attachment mode. The inline mode would display images in the browser rather than downloading them. This is typically used when viewing file details or previewing.
  • Upload Validation Token: Code review shows the GetFileUploadToken endpoint exists for generating upload validation tokens. This is used internally for secure uploads but is not directly exposed in the UI testing.
  • Drag and Drop: The upload dialog supports drag-and-drop functionality as indicated by the "Drag and Drop a File or Select an Option" text, though this was tested programmatically rather than via actual drag-drop interaction.
  • ZIP File Handling: The upload dialog includes a "Decompress Zip Files" toggle option, allowing users to automatically extract ZIP archives during upload.
  • File Workflow: The code includes workflow support for files (draft, submit, approve states) but this requires specific workflow configuration on the folder.
