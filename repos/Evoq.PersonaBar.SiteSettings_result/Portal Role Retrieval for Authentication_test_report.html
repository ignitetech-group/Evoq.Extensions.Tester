Observations

  • Code Analysis: The GetDnnPortalRoles method in Components/EvoqSiteSettingsController.cs (line 127-131) retrieves roles using RoleController.Instance.GetRoles(portalId) and returns only the role names.
  • API Structure: The API endpoint in Services/EvoqSiteSettingsController.cs (line 118-130) accepts an optional portalId parameter, defaulting to the current portal if not specified.
  • System Roles Included: Both system roles (Administrators, Registered Users, Unverified Users, Subscribers) and custom roles (Community Manager, Content Editors, etc.) are included in the results.
  • UI Integration: The roles dropdown is part of the URL path configuration form and is used for DNN impersonation during crawling operations.
