Test 1: View List of Excluded Directories

Status: PASS

What was tested

Verify that the list of excluded directories is displayed correctly with directory paths.

Steps taken

  1. Logged in as SuperUser (host)
  2. Navigated to Site Settings > Search > Crawling
  3. Scrolled to the "Excluded Directories" section
  4. Verified the directory list is visible with paths

Result

The Excluded Directories section displays correctly with:

  • Directory header column
  • List of excluded directories (Badges/, Templates/)
  • Delete icons for each directory
  • "Add Directory" button available

Test 3: Add Duplicate Directory (Error Scenario)

Status: PASS

What was tested

Verify that the system shows an appropriate error when attempting to add a directory that is already in the excluded list.

Steps taken

  1. With the Add Directory form open, selected "Badges" folder
  2. Clicked "Save" button (Badges/ already exists in excluded list)
  3. Observed error notification

Result

The system correctly validates for duplicates:

  • Error notification displayed: "Directory duplicated. This directory 'Badges/' is already added"
  • Duplicate directory was NOT added to the list
  • Server responded with 500 status code indicating validation failure

Screenshot: Badges selected (already exists in excluded list)

Test 4: Remove Directory from Excluded List

Status: PASS

What was tested

Verify that a directory can be removed from the excluded list with proper confirmation.

Steps taken

  1. Located the delete (trash) icon next to "PermissionsTestFolder"
  2. Clicked the delete icon
  3. Confirmation dialog appeared: "Are you sure you want to delete PermissionsTestFolder?"
  4. Clicked "Yes" to confirm deletion
  5. Verified the directory was removed from the list

Result

The delete functionality works correctly:

  • Delete icon triggers a confirmation dialog
  • Confirmation dialog shows the directory name being deleted
  • User can choose "No" to cancel or "Yes" to confirm
  • Clicking "Yes" removes the directory from the list
  • List updates immediately to reflect the deletion

Screenshot: Confirmation dialog asking to delete PermissionsTestFolder

Screenshot: Excluded Directories after deletion (only Badges/ and Templates/ remain)

Observations

  • Folder Picker Usability: The folder picker displays a tree structure of all available folders, making it easy to navigate and select directories. However, clicking on folders can be tricky due to UI overlay issues - the selection sometimes requires precise clicking on the folder name.
  • Error Handling: The duplicate directory validation works well and provides a clear error message. The error notification appears as a toast message but disappears quickly.
  • Confirmation Dialog: The delete confirmation dialog is well-designed with a warning icon and clear Yes/No options, preventing accidental deletions.
  • Directory Display: Directories are shown with trailing slashes (e.g., "Badges/") which clearly indicates they are folders.
  • Code Reference: The feature is implemented in EvoqSiteSettings.Web/src/components/CrawlingSettings/DirectoryList/index.jsx using a FolderPicker component and API endpoints in Services/EvoqSiteSettingsController.cs.
