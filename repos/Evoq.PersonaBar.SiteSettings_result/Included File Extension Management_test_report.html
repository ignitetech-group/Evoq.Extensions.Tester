Test Scenarios

1. View List of All Included Extensions

PASS

Objective: Verify that the list of included file extensions is displayed correctly with all relevant information.

  1. Navigate to Admin > Settings > Site Settings
  2. Click on the "Search" tab
  3. Click on the "File Extensions" sub-tab
  4. Observe the "Included File Extensions" section

Result: The list displays correctly showing:

  • File Extension column (e.g., .doc, .docx, .ppt, .pptx, .txt, .xls, .xlsx)
  • File Type column showing IFilter handler name (e.g., "Microsoft Office Filter", "Plain Text filter")
  • Content Crawling status (checkmark for available, warning icon for unavailable)
  • Delete icon for each extension

Screenshot showing the File Extensions list with all columns

2. Add New File Extension - Whitelist Validation

PASS

Objective: Verify that adding a file extension not in the whitelist shows an error.

  1. Click "Add File Type" button
  2. Enter "pdf" in the File Extension field
  3. Click Save
  4. Observe the validation error

Result: The system correctly validates that "pdf" is not in the allowed whitelist and displays an error indicator. The extension is not added.

Add File Extension form opened

Error shown when trying to add "pdf" (not in whitelist)

Observation: The whitelist validation correctly prevents adding extensions that are not allowed. Extensions like "pdf" and "htm" are not in the allowed list - only specific file types with IFilter support appear to be allowed.

5. Add Extension with Leading Dot

PARTIAL

Objective: Verify handling of extensions entered with a leading dot (e.g., ".doc" instead of "doc").

  1. Click "Add File Type" button
  2. Enter ".doc" (with leading dot) in the File Extension field
  3. Click Save
  4. Observe the result

Result: The frontend validation accepts ".doc" with a leading dot, but the server returns a 500 error because ".doc" already exists (duplicate). The system handles the leading dot by normalizing it server-side.

Extension entered with leading dot - passed frontend validation

Observation: The server correctly identifies ".doc" as a duplicate of the existing ".doc" extension, suggesting it normalizes the extension format. However, a 500 Internal Server Error is returned rather than a user-friendly error message. This could be improved to show the same duplicate error message displayed for other duplicates.

6. Delete Included File Extension

PASS

Objective: Verify that deleting an extension shows a confirmation dialog and works correctly.

  1. Click the trash icon next to ".xlsx" extension
  2. Observe the confirmation dialog
  3. Click "No" to cancel (or "Yes" to confirm deletion)

Result: The system correctly shows a confirmation dialog asking "Are you sure you want to delete .xlsx?" with "No" and "Yes" options. Clicking "No" cancels the operation, clicking "Yes" would delete the extension.
