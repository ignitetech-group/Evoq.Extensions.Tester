Tags with special characters: @, #, &, _ all accepted

Result:

Special characters in tags are properly supported:

  • special@tag - @ symbol accepted, URL encoded as %40
  • tag#test - # symbol accepted, URL encoded as %23
  • test&tag - & symbol accepted, URL encoded as %26
  • under_score - underscore accepted without encoding

Test 7: Tag Case Sensitivity

PASSED

Steps Performed:

  1. Noted existing "wiki" tag in tag cloud with count of 3
  2. Created article "Case Sensitivity Test Article"
  3. Entered uppercase tag "WIKI"
  4. Published the article
  5. Checked how the tag was stored and displayed
  6. Verified tag cloud count for "wiki"
Entering uppercase "WIKI" tag in article creation form
Result: Tag stored as lowercase "wiki"
Tag cloud shows "wiki" count increased from 3 to 4

Result:

Tags are CASE-INSENSITIVE. Key findings:

  • Entered "WIKI" (uppercase) in the form
  • System normalized it to "wiki" (lowercase)
  • Tag cloud "wiki" count increased from 3 to 4
  • Uppercase and lowercase tags are treated as the same tag

This is expected behavior for tag systems to prevent duplicate tags with different casing.
