Code Review Summary

Reviewed the following key files:

  • ReportController.cs - Contains SubmitReport and SubmitCommentReport API endpoints
  • Notifications.cs - Handles ReportArticle notification delivery to moderators
  • dnn.wiki.Article.js - Client-side JavaScript for article interactions including report functionality
  • Article.ascx - Contains the Report link bound to knockout.js click handler

The reporting flow: User clicks Report link -> Dialog shows with 4 reason options -> API call to SubmitReport -> Notification sent to moderators via SocialNotificationsController -> Scoring points logged for reporter and author

Scenario 5: Report Comment

NOT TESTED

Reason:

No comments were available on the wiki articles in the test environment. The comment section was empty on all tested articles.

Wiki article page showing empty comments section

Code Analysis:

Based on code review, comment reporting is implemented via the SubmitCommentReport endpoint in ReportController.cs. The flow is similar to article reporting:

  • Comment is validated using Journal.Instance.GetComment(report.ReferenceId)
  • Parent article is retrieved for context
  • Notification is sent to moderators with comment content
  • Scoring points are logged for FlaggedComment action

Prerequisite for Testing: Create comments on wiki articles before testing this scenario.
