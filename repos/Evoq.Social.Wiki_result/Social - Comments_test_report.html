Feature Overview

The Social - Comments feature allows users to interact with wiki articles through comments. Key functionalities include:

  • Post new comments on wiki articles
  • Edit own comments
  • Delete own comments
  • Reply to existing comments (nested comments)
  • @mention users in comments
  • Like and report comments
  • Comment moderation (approve/reject)
  • Comment notifications

Observations

Features Verified in Code but Requiring Additional Setup to Test:

  • Comment Moderation (approve/reject): Code in SocialController.cs shows ApproveComment and DeleteComment methods with CanModerateComments() permission checks. This feature requires moderation to be enabled in module settings. The code handles SendCommentAwaitingApprovalNotifications and RemoveCommentAwaitingApprovalNotifications.
  • Profanity Filter: Based on Activity Stream testing, profanity filtering is handled at the SocialLibrary level. Configuration is managed at the site level.
  • Comment Notifications: Code in Notifications.cs shows NewCommentNotifications and UserMentionNotificationsController for sending notifications when comments are posted. During testing, the notification count changed (visible in sidebar), indicating notifications are being generated.
  • Comment Permissions: Code shows CanModerateComments() checking for Edit or Moderate permissions. Full permission testing would require logging in as users with different role assignments.

Additional Observations:

  • Comments display with username as "Anonymous" due to display name configuration
  • Edit/Remove actions are available via pencil icon dropdown menu
  • Comments support Like, Reply, and Report actions
  • Comment sorting (Newest/Oldest) appears when multiple comments exist
  • User points increased after posting comments (visible in sidebar: 85 to 90 points)
