Critical Bug Found

BUG: Community Analytics Dashboard Fails to Initialize

Severity: Critical (Blocks entire feature)

Error Message:

TypeError: utility.serializeCustomDate is not a function at dashboardClass.init (social-dashboard-combined-instance.js:1025:39) at Object.init (social-dashboard-combined.js:16:23) at extension.js:104:32

Impact: The entire Community Analytics panel fails to render. Users cannot access any analytics data for Discussions, Blogs, Answers, Ideas, Wiki, or Social Events.

Root Cause: The utility.serializeCustomDate function is being called but does not exist in the utility object. This appears to be a missing dependency or incompatibility issue.

Code Analysis

The following code files were reviewed:

Services/CommunityAnalyticsController.cs

The API controller provides endpoints for analytics data retrieval:

  • GetDashboardInfo - General dashboard metrics
  • GetModuleDashboardInfo - Module-specific dashboard (e.g., Discussions)
  • GetModulePopularContent - Popular content for a specific module
  • GetModuleTagStats - Tag statistics per module
  • GetModuleContextData - Context data for modules

The server-side API appears to be properly implemented. The issue is with the client-side JavaScript initialization.

Evoq.PersonaBar.CommunityAnalytics.dnn

The manifest correctly defines extensions for:

  • Answers (path: social-dashboard?Answers)
  • Blogs (path: social-dashboard?Blogs)
  • Discussions (path: social-dashboard?Discussions)
  • Social Events (path: social-dashboard?Social Events)
  • Ideas (path: social-dashboard?Ideas)
  • Wiki (path: social-dashboard?Wiki)

Recommendations

  1. Immediate Fix Required: Investigate the utility.serializeCustomDate function. Either:
    • Add the missing function to the utility module
    • Update the dependency to include the required function
    • Check for version mismatches between PersonaBar components
  2. Error Handling: Add try-catch blocks around initialization to provide graceful degradation and meaningful error messages to users
  3. Regression Testing: After fix, run full test suite for all Community Analytics tabs (Discussions, Blogs, Answers, Ideas, Wiki, Social Events)
