Observations

  • Code Review: The code in Services/CommunityAnalyticsController.cs shows proper API endpoints for module-specific analytics (GetModuleDashboardInfo, GetModuleTagStats, GetModulePopularContent) that accept date range parameters.
  • Extension Configuration: The Evoq.PersonaBar.CommunityAnalytics.dnn file correctly defines the Answers extension with path social-dashboard?Answers.
  • Root Cause: The JavaScript error utility.serializeCustomDate is not a function suggests that the utility module is either not loaded, not properly initialized, or the serializeCustomDate function is missing from the utility object. This is likely a module loading order issue or a missing dependency.
  • Affected Components: The error occurs in social-dashboard-combined-instance.js at line 1025, during the dashboardClass.init() call.
  • Site Analytics Works: Note that Site Analytics (under Dashboard > Site Analytics) does load, suggesting the issue is specific to the Community Analytics module initialization.

Recommendations

  1. Investigate Missing Function: Check if utility.serializeCustomDate is defined in the PersonaBar utility module and ensure it's properly exported/available.
  2. Check Module Dependencies: Verify that all required JavaScript modules are loading in the correct order before Community Analytics initializes.
  3. Review Recent Changes: If this worked previously, review recent changes to the utility module or Community Analytics scripts.
  4. Browser Console: The full error stack trace is available in browser console for debugging.
