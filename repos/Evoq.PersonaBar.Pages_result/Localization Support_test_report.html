Test 2: Language Switching / Locale Awareness PASS

Objective: Verify that the localization system responds to language/locale changes and properly falls back to default language when language-specific resources are not available.

Steps Taken:
  1. Switched the site locale from English (en-US) to German (de-DE)
  2. Navigated to Content > Pages in the PersonaBar
  3. Compared the UI display between English and German locales
  4. Verified localization behavior for date formatting

Expected Result:

  • Date formatting should change based on locale
  • UI strings should fallback to English when German resources are not available

Actual Result: Localization system works correctly:

Aspect English Locale (en-US) German Locale (de-DE) Status
Date Format 12/27/2025 (MM/DD/YYYY) 27.12.2025 (DD.MM.YYYY) Locale-aware
UI Labels English English (fallback) Expected behavior

Key Finding: The date format changed from US format (MM/DD/YYYY) to German format (DD.MM.YYYY), demonstrating that the localization system is locale-aware. UI labels remain in English as expected since no German-specific resource files exist.

Observations

  • Resource File Location: The localization resources are stored in admin/personaBar/App_LocalResources/EvoqPages.resx
  • Single Language Resource: Only the default English resource file exists. No language-specific files (e.g., EvoqPages.de-DE.resx) were found in the codebase.
  • Fallback Behavior: The localization system correctly falls back to the default English strings when language-specific resources are not available.
  • Date/Time Localization: While UI string translations are not available for other languages, date/time formatting is properly locale-aware.
  • Resource Keys: The EvoqPages.resx file contains keys for workflow-related strings (WorkflowTitle, WorkflowRunning), template strings (Template, TemplateName, SaveAsTemplate), and filter options (FilterByPublishDateText, FilterbyWorkflowText).
