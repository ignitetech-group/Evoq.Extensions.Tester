Activity Stream Filtering - Test Report

Feature Information

Extension: Evoq.Social.ActivityStream (Module)

Feature Name: Activity Stream Filtering

Description: Filter activity stream by various criteria and activity types

Feature Priority: High

UI Location: Activity Stream > Filter Options

Test Date: December 30, 2025

Tester: Automated Test (Claude)

Test Scenarios

1. Filter by Activity Type PASS

Steps Taken:
  1. Navigated to Community page where Activity Stream is displayed
  2. Located the "Filter by activity" dropdown
  3. Selected "Photos" from the dropdown
  4. Verified only photo posts are displayed
  5. Selected "Posts" from the dropdown
  6. Verified only text posts are displayed
  7. Selected "Added to Wiki" from the dropdown
  8. Verified only wiki articles are displayed

Screenshot: Activity Stream filtered by Photos - showing only photo posts

Screenshot: Activity Stream filtered by Posts - showing only text posts

Screenshot: Activity Stream filtered by Added to Wiki - showing only wiki articles

Result: The filter dropdown correctly filters the activity stream by the selected activity type. Multiple activity types were tested (Photos, Posts, Added to Wiki) and all worked as expected.

2. Clear All Filters PASS

Steps Taken:
  1. With a filter active, selected "Filter by activity" (default option)
  2. Verified that all activity types are now displayed

Screenshot: Activity Stream with filter cleared - showing all activity types

Result: Selecting the default "Filter by activity" option successfully clears any active filters and shows all activity types.

3. Save Filter Preferences (Create New Feed) PASS

Steps Taken:
  1. Located "Activity Stream Filter" section on the right sidebar
  2. Clicked on "Create New Feed" link
  3. Custom Feed Builder dialog opened
  4. Entered filter name: "Photos and Wiki Filter"
  5. Selected activity types: Photos, Added to Wiki
  6. Clicked Save button
  7. Verified the new filter appears in the Activity Stream Filter section

Screenshot: Custom Feed Builder dialog with filter options

Screenshot: Filter configured with name and activity types selected

Screenshot: New filter "Photos and Wiki Filter" saved and visible in sidebar

Result: Custom filters can be created and saved successfully using the Custom Feed Builder. The saved filter appears in the Activity Stream Filter section for future use.

4. Apply Saved Filter PASS

Steps Taken:
  1. Located the saved "Photos and Wiki Filter" in the Activity Stream Filter section
  2. Clicked on the filter name
  3. Verified that only Photos and Wiki articles are displayed

Screenshot: Saved filter applied - showing only Photos and Wiki articles

Result: Saved custom filters can be applied by clicking on them. The activity stream correctly shows only the activity types configured in the saved filter.

5. Filter by User NOT TESTED

Observations:

The "Activity Authors" field is available in the Custom Feed Builder dialog. This allows users to filter activities by specific authors. The field includes an autocomplete search feature ("Add an Author"). This feature is available when creating custom feeds but was not tested in this session due to time constraints.

Result: Feature is available through Custom Feed Builder but not tested in this session.

6. Filter by Group NOT TESTED

Observations:

The "Activity Groups" field is available in the Custom Feed Builder dialog. This allows users to filter activities by specific groups. The field includes an autocomplete search feature ("Add a Group"). This feature is available when creating custom feeds but was not tested in this session due to time constraints.

Result: Feature is available through Custom Feed Builder but not tested in this session.

7. Filter by Date Range NOT AVAILABLE

Observations:

Based on the code review and UI inspection, there is no explicit date range filter available in the current Activity Stream implementation. The activity stream displays items chronologically, but there is no UI control to filter by specific date ranges.

Result: Date range filtering is not available in the current implementation.

8. Apply Multiple Filters PASS

Observations:

Multiple filters can be applied through the Custom Feed Builder by selecting multiple activity types. The "Photos and Wiki Filter" test demonstrated this capability by selecting both "Photos" and "Added to Wiki" activity types.

Result: Multiple activity types can be combined in a single custom filter through the Custom Feed Builder.
