BUG FOUND: Cannot Delete Posts with File Attachments

Severity: High

Steps to Reproduce:

  1. Create a post with a file attachment (e.g., .txt, .pdf, .docx)
  2. Click the delete button on the post
  3. Confirm deletion in the dialog
  4. Observe 400 Bad Request error

Expected Result: Post with attachment should be deleted successfully

Actual Result: Server returns 400 Bad Request, post remains in feed

Error Details: Alert displays "undefined" indicating the server response error message was not properly parsed

Possible Cause: The SoftDelete method in ActivityStreamServicesController.cs may not properly handle journal items with file attachments, or there may be a foreign key constraint issue when attempting to delete associated file records.

Code Reference: ActivityStreamServicesController.cs:172 - SoftDelete method
