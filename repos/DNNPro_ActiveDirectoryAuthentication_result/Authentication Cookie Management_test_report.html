Result

This confirms:

  • Authentication cookies have been completely cleared
  • The server no longer recognizes the user as authenticated
  • Protected resources are properly secured after logout

Observations

The following observations were made during testing regarding features that exist in code but are not directly testable via UI:

  • Cookie Timeout Configuration: The code at AuthenticationController.cs:121-141 reads a PersistentCookieTimeout setting from web.config to customize cookie expiration. This configuration is managed at the server level (web.config) and is not exposed in the UI.
  • Authentication Status Cookie: The GetStatus and SetStatus methods (lines 661-729) manage an authentication status cookie (AUTHENTICATION_STATUS_KEY) that tracks Windows vs. Forms authentication state. This is used internally and not visible in UI.
  • GetAuthCookieTimeout: The method at lines 834-846 reads the forms authentication timeout from web.config's system.web/authentication/forms section. Default is 30 minutes if not specified.
  • HttpOnly Cookies: Authentication cookies are properly configured as HttpOnly (verified by inability to read via JavaScript), which is a security best practice to prevent XSS attacks from stealing session cookies.
  • Windows Authentication: The Windows Login tab on the login page was not fully testable due to an IIS configuration error (500.19) on the WindowsSignin.aspx endpoint. Standard authentication was tested successfully.
