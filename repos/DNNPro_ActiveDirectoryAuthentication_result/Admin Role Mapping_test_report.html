Code Analysis Summary

Based on code review of the relevant source files:

How Admin Role Mapping Works

  1. Configuration Storage: The Admin Role Mapping value is stored in portal settings as "AD_AdminRoleMapping"
  2. During User Login: The AddUserRoles() method in UserController.cs retrieves the user's AD groups
  3. Role Assignment Logic:
    • If the user belongs to the mapped AD group AND is NOT currently in the admin role, they are added to the admin role
    • If the user does NOT belong to the mapped AD group AND IS currently in the admin role, they are removed from the admin role
  4. Super User Protection: The code only affects regular administrator role, not super user permissions
// From UserController.cs - ManageAdminRoleMapping method (lines 212-230) private static void ManageAdminRoleMapping(int portalId, int adminRoleId, bool existInRoleMapping, UserInfo user) { var adminRole = RoleController.Instance.GetRoleById(portalId, adminRoleId); if (adminRole != null) { if (existInRoleMapping) { if (!user.IsInRole(adminRole.RoleName)) { RoleController.Instance.AddUserRole(portalId, user.UserID, adminRole.RoleID, RoleStatus.Approved, false, DateTime.Today, Null.NullDate); } } else if (user.IsInRole(adminRole.RoleName)) { RoleController.DeleteUserRole(user, adminRole, PortalSettings.Current, false); } } }
