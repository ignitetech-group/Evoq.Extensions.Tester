IP-Based Auto Authentication - Test Report

Feature Information

Feature Name IP-Based Auto Authentication
Description Automatically authenticate users based on their IP address ranges
Feature Priority Medium

Test Date December 30, 2025
Tester Automated Test (Claude)

Code Analysis Summary

Based on code review of AuthenticationModule.cs and Settings.ascx.cs:

  • IP addresses are stored in config.AutoIP and split by semicolons (;)
  • Supports single IPs (e.g., 192.168.1.100) and IP ranges with dash (e.g., 192.168.1.1-192.168.1.254)
  • IP validation requires exactly 4 octets (3 dots)
  • Uses IpAddressToLong() function for range comparison
  • Client IP retrieved via Utilities.GetIP4Address(request.UserHostAddress)

Test Scenarios

Test 1: Configure Single IP Address PASS

Objective: Verify that a single IP address can be configured for auto-authentication.

Steps Taken:
  1. Navigated to Settings > Extensions > Authentication Systems
  2. Selected DNNPro_ActiveDirectoryAuthentication
  3. Clicked on Site Settings tab
  4. Entered 192.168.1.100 in the Auto-login IP Address field
  5. Clicked "Update Authentication Settings"

Result: PASS - Single IP address was saved successfully.

Test 4: IP Address Format Validation PASS

Objective: Verify that invalid IP addresses are rejected with appropriate error message.

Steps Taken:
  1. Entered an invalid IP address: 192.168.1 (only 3 octets instead of 4)
  2. Clicked "Update Authentication Settings"
  3. Observed the validation error message

Result: PASS - Validation correctly rejected the invalid IP with error message "192.168.1 is not a valid IP address".

Invalid IP address (192.168.1) entered - missing fourth octet

Validation error displayed: "192.168.1 is not a valid IP address"

Observations

  • LDAP Connection Errors: LDAP connection errors are displayed when saving settings. This is expected behavior in a test environment without an actual Active Directory server. The IP-Based Auto Authentication settings are saved regardless of LDAP connectivity.
  • IIS Configuration Issue: Initial navigation triggered a 500.19 error due to Windows Authentication configuration in IIS. This was bypassed by navigating directly to the login page.
  • Field Behavior: The Auto-login IP Address field accepts multi-line input and trims trailing semicolons automatically.
