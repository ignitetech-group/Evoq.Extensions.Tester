Code Review Findings

Based on code review, the Default Template Management feature implements the following logic:

1. Default Template Identification (useDefaultSkin property)

// TemplatesController.cs - Lines 295-298 private bool UseDefaultSkin(TabInfo tab) { return !string.IsNullOrEmpty(tab.SkinSrc) && tab.SkinSrc.Equals(PortalSettings.DefaultPortalSkin, StringComparison.InvariantCultureIgnoreCase); }

The default template is identified by comparing the page's SkinSrc with the portal's DefaultPortalSkin setting.

2. Auto-selection When Creating New Template

// templatesCreateManager.js - Lines 153-159 getDefaultSelectedTemplate = function() { var defaultSelected = defaultTemplate; if (!defaultSelected && viewModel.pageTemplates().length > 0) { defaultSelected = viewModel.pageTemplates()[0]; } return defaultSelected; };

When creating a new template, the default template is automatically selected. If no default exists, the first template is used.

3. Delete Protection for Default Template

// TemplatesController.cs - Lines 256-259 if (_pageTemplatesController.IsTemplatePage(tab) && UseDefaultSkin(tab)) { return Request.CreateErrorResponse((HttpStatusCode)UnprocessableEntity, Localization.GetString("PageTemplate_Cannot_Delete_Default_Template", Dnn.PersonaBar.Library.Constants.SharedResources)); }

The system prevents deletion of the default template by returning a 422 (Unprocessable Entity) error with the message "PageTemplate_Cannot_Delete_Default_Template".

Appendix: Feature Logic Flow

Default Template Management Flow:

1. Template Loading (templatesCreateManager.js:123-141)
   - GET /API/Templates/GetPageTemplates
   - For each template: check useDefaultSkin property
   - Template with useDefaultSkin=true marked as defaultTemplate

2. New Template Creation (templatesCreateManager.js:161-173)
   - Call getDefaultSelectedTemplate()
   - Auto-select default template (or first if none)
   - Set templateId for new template form

3. Template Deletion (TemplatesController.cs:248-269)
   - Check if page is template AND uses default skin
   - If true: return 422 "Cannot_Delete_Default_Template"
   - If false: proceed with soft delete

4. Default Skin Check (TemplatesController.cs:295-298)
   - Compare tab.SkinSrc with PortalSettings.DefaultPortalSkin
   - Case-insensitive comparison
   - Returns boolean indicating default status
