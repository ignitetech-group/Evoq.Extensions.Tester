Observations

  • API Implementation: The GetUrlOptions API endpoint (found in UrlManagementServiceController.cs) returns Locales, StatusCodes, SiteAliases, PrimaryAliasId, HasParent, and SiteAliasUsages - all of which were successfully verified through the UI.
  • Multi-Locale Support: The system correctly supports 5+ locales with separate site aliases for each language.
  • Status Codes: Only two status codes are supported (200 Active and 301 Redirect) as defined in the code.
  • Primary Alias: The primary alias identification works correctly, with the base alias (localhost:8081) serving as the default, and each locale having its own primary alias.
  • Site Alias Mapping: The UI provides three mapping modes (Canonical, Redirect, None) which corresponds to the PortalAliasUsageType options in the API.
