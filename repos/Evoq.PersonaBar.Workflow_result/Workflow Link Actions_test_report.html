Technical Background

The Workflow Link Actions feature is an internal backend component that generates links used in workflow notifications. These links allow users to quickly navigate to content (pages or files) that require their attention in the workflow process.

API Endpoint

/DesktopModules/EvoqContentLibrary/API/WorkflowsLinks/Index/?id={id}&type={type}

Where:

  • id = Page ID (TabID) or File ID
  • type = Tab for pages or File for files

Observations

  • Internal Feature: Workflow Link Actions is an internal/backend feature with no direct UI to generate links. Links are created programmatically when workflow notifications are sent.
  • Link Generation: Links are generated by WorkflowActionHelper.GetWorkflowLink() method and embedded in workflow notifications via:
    • WorkflowStartTabAction - When page workflow starts
    • StateCompleteTabAction - When workflow state completes
    • StateDiscardTabAction - When workflow state is discarded
    • NotificationTemplateHelper - For file workflow notifications
  • DAM Module Dependency: File workflow links require a Digital Asset Management (DAM) module to be installed. Without it, file links return "DAM module not found".
  • Edit Mode: Page workflow links automatically set the user to Edit mode (SetUserModeToEditMode) and store the last page cookie for better UX.
  • Mobile Support: The code includes separate handling for mobile devices via IsMobile() check, providing version-specific URLs for mobile users.
  • Link Expiration: No explicit link expiration was found in the code - links remain valid as long as the content exists and user has permissions.
