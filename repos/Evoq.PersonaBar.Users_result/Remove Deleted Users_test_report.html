Test Cases

Test 1: Display Confirmation Dialog Before Removal

Status: PASS

Description: Verify that a confirmation dialog is displayed before removing deleted users.

Steps:
  1. Navigate to Users section (Manage > Users)
  2. Click the "Remove Deleted Users" button
  3. Observe that a confirmation dialog appears

Expected Result: A confirmation dialog with warning message should appear with Yes/No options.

Actual Result: Confirmation dialog displayed with message "Do you wish to permanently remove deleted users? You cannot undo this operation." with "No" and "Yes" buttons.

Test 2: Test with No Deleted Users Present

Status: PASS

Description: Verify behavior when there are no deleted users to remove.

Steps:
  1. Filter users to show "Deleted" users
  2. Verify no deleted users are present
  3. Click "Remove Deleted Users" button
  4. Confirm the removal in the dialog

Expected Result: Success notification should appear even when no deleted users exist.

Actual Result: Success notification "All deleted users have been removed" was displayed. The operation completed successfully without errors.

Test 3: Verify Only Affects Users with Deleted Status

Status: PASS

Description: Verify that the removal only affects users who have been soft-deleted.

Steps:
  1. Create a new test user "DeleteTest User"
  2. Soft delete the user using "Delete User" from user menu
  3. Verify user appears in "Deleted" filter
  4. Verify user no longer appears in "Authorized" filter

Expected Result: Only users with deleted status should be affected by the removal.

Actual Result: Created user "DeleteTest User" was successfully soft-deleted and appeared only in the "Deleted" filter, confirming the delete status was properly applied.

Test 4: Successfully Remove All Deleted Users

Status: PASS

Description: Verify that clicking "Remove Deleted Users" permanently removes all deleted users.

Steps:
  1. With a deleted user present, click "Remove Deleted Users"
  2. Confirm the action by clicking "Yes"
  3. Observe the result

Expected Result: All deleted users should be permanently removed from the system.

Actual Result: The deleted user "DeleteTest User" was permanently removed from the system.

Test 5: Verify User List Refreshes After Removal

Status: PASS

Description: Verify that the user list automatically refreshes after removal to show updated state.

Steps:
  1. After confirming removal, observe the user list
  2. Verify that the deleted user is no longer visible
  3. Verify that "No users found." message appears in Deleted filter

Expected Result: The user list should automatically refresh to show "No users found."

Actual Result: After removal, the user list immediately refreshed and displayed "No users found." in the Deleted filter.

Test 6: Display Success Notification After Completion

Status: PASS

Description: Verify that a success notification is displayed after removing deleted users.

Steps:
  1. Complete the "Remove Deleted Users" operation
  2. Observe the notification that appears

Expected Result: A success notification should appear confirming the operation completed.

Actual Result: Success notification "All deleted users have been removed" was displayed after the operation completed.

Note: The success notification is transient and disappears after a few seconds. It was captured in the page state during testing.

Observations

  • Admin-Only Permission: Code review of permissionHelpers.js confirms that the "Remove Deleted Users" button is only visible to admin users. The function canRemoveDeletedUsers(settings) returns settings.isAdmin, meaning only users with admin privileges can see and use this feature. This is a security feature to prevent unauthorized permanent deletion of user data.
  • Confirmation Dialog: The confirmation dialog provides clear warning that the operation cannot be undone, which is good UX for destructive operations.
  • Transient Notifications: Success notifications appear briefly and auto-dismiss. While functional, screenshots may not capture them if taken after they disappear.
  • Graceful Handling: The feature handles the case of no deleted users gracefully - it still shows the success message without errors.
  • List Refresh: The user list automatically refreshes after removal, ensuring the UI reflects the current state without requiring manual refresh.
