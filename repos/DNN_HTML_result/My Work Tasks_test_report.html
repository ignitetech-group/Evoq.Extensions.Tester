Observations

  • Module Definition Exists: The My Work control is properly registered in the module manifest (dnn_HtmlPro.dnn) with controlKey="MyWork" and controlSrc="DesktopModules/HTML/MyWork.ascx"
  • Missing Physical File: The actual control file MyWork.ascx is not deployed to the server at the expected location
  • Documentation Exists: Documentation for the My Work feature exists at Documentation/Web Content Management/HTML Pro/Module Editors/Managing My Workflow Tasks.html, describing how to use the feature
  • Possible Cause: This could be a deployment issue where the control file was not included in the module package, or the file was removed during an upgrade
  • Recommendation: The MyWork.ascx file needs to be created/restored and deployed to the server at /DesktopModules/HTML/MyWork.ascx
