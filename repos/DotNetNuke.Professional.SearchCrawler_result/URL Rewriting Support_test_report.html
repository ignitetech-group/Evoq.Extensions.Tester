Feature Description

The URL Rewriting Support feature handles friendly URLs and URL rewriting during crawling operations. This ensures that the SearchCrawler module correctly processes both friendly URLs (e.g., /Our-Products) and traditional query string URLs (e.g., /default.aspx?TabId=22), avoiding duplicate content indexing.

Technical Observations

Code Analysis Findings:

  • UrlRewriter.cs: The RewriteUrl() method loads rules from SiteUrls.config and handles reserved pages (login, register, terms, privacy) with special rewriting logic.
  • Spider.cs: The addURI() method checks the UseFriendlyUrls host setting and calls UrlRewriter.RewriteUrl() to normalize URLs before crawling.
  • Duplicate Prevention: The crawler uses regex patterns to identify and deduplicate URLs that may appear in both friendly and query string formats.
  • Language Support: URL rewriting automatically extracts language codes from URLs and adds them as parameters.

Dependencies

This feature depends on the URL Rewriting Module being properly configured in DNN. The SEO settings must have "Redirect to Friendly URLs" enabled for full functionality.
