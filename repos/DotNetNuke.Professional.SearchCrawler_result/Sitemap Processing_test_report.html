Sitemap Processing - Test Report

Extension: DotNetNuke.Professional.SearchCrawler

Feature: Sitemap Processing

Test Date: December 30, 2025

UI Location: Admin > Site Settings > Search > Crawling

Feature Description

The Sitemap Processing feature processes XML sitemaps to seed URL crawling operations. When a sitemap URL is configured for a crawl path, the system downloads and validates the sitemap, extracts all URLs, and uses them to seed the web crawler instead of starting from just the base URL.

Key Code Files

SitemapHelper.cs - Handles sitemap downloading, validation, and URL extraction
Spider.cs - Uses SitemapHelper to seed initial URLs for crawling

Test 6: Sitemap Index File Processing CODE VERIFIED

Code review confirms the system supports sitemap index files.

SitemapHelper.cs (Lines 106-116):
The code checks for <sitemapindex> elements and recursively processes nested sitemaps: 
if (_sitemapDocument.DocumentElement.SelectSingleNode("//sm:sitemapindex", nm) != null)
{
    foreach (XmlNode sitemapNode in _sitemapDocument.DocumentElement.SelectNodes("//sm:loc", nm))
    {
        var sh = new SitemapHelper(sitemapNode.InnerText);
        if (sh.IsValid)
        {
            _urls.AddRange(sh.Urls);
        }
    }
}
Implementation: When a sitemap index is detected, the system creates new SitemapHelper instances for each referenced sitemap and recursively extracts all URLs.

Test 7: Gzip Compressed Sitemap Support CODE VERIFIED

Code review confirms the system supports gzip-compressed sitemaps (.gz files).

SitemapHelper.cs (Lines 65-67):
sitemapString = SitemapUrl.ToLower().EndsWith(".gz")
    ? Encoding.UTF8.GetString(UnGzip(wc.DownloadData(SitemapUrl)))
    : wc.DownloadString(SitemapUrl);
Implementation: The system automatically detects .gz file extensions and decompresses the sitemap before parsing.

Conclusion

The Sitemap Processing feature is fully functional. All tested scenarios passed successfully:

  • UI Configuration: The Sitemap URL field is accessible in the Crawling settings
  • Validation: Invalid sitemaps are rejected with clear error messages
  • Valid Sitemaps: Properly formatted sitemaps are accepted and configurations are saved
  • Code Features: Sitemap index support, gzip compression, and XSD schema validation are implemented

Technical Implementation Notes

  • Sitemap validation occurs at save time (not runtime)
  • The Spider.Start() method uses SitemapHelper.Urls to seed crawling when a sitemap URL is configured
  • If sitemap validation fails, the system falls back to crawling from the base URL only
  • Schema validation uses official sitemaps.org XSD schemas
