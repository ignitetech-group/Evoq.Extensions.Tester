Social sharing options with Twitter disabled and RSS enabled

Test 5: Save Settings and Verify Persistence PARTIAL

Objective: Verify that settings are saved and persist after page refresh

Steps Taken:

  1. Made changes: Workflow to "Direct Publish", Twitter disabled, RSS enabled
  2. Clicked the Save button
  3. Settings panel closed automatically
  4. Refreshed the page
  5. Reopened the settings panel to verify persistence

After clicking Save - panel closed

Settings panel after page refresh

Issue Found:

After page refresh, the settings appeared to revert to default values:

  • Workflow showed "Content Approval" instead of saved "Direct Publish"
  • All social sharing options appeared enabled

Possible causes: Caching issue, API save not completing, or front-end state management issue.

Result: Save button worked and closed the panel, but settings persistence needs further investigation.
