Test 4: GetRoles Endpoint

FAIL

Objective

Test the GetRoles endpoint returns list of portal roles.

Endpoint

GET /DesktopModules/DNNCorp/GoogleAnalyticsConnector/API/Services/GetRoles

Steps Taken

  1. Navigate to the endpoint while authenticated
  2. Verify response

Results

The endpoint returns HTTP 500 Internal Server Error with message "An error has occurred."

Issue Analysis

Looking at the code, the potential issue is in line 53:

data.Add(new { RoleId = Int32.Parse(Globals.glbRoleAllUsers), RoleName = Globals.glbRoleAllUsersName });

The Int32.Parse(Globals.glbRoleAllUsers) call may be failing if the constant value is not a valid integer string, or there may be an issue with accessing PortalSettings in the API context.

Screenshot: 500 Internal Server Error from GetRoles endpoint

Recommendation

Debug the GetRoles method to identify the exact exception. Consider adding try-catch error handling and logging to diagnose the issue.
