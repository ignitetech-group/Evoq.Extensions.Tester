Observations

  • No Format Validation: The connector does not validate the tracking ID format. It accepts any non-empty string, including UA- format, G- format, invalid formats, and special characters. The code only uses the format prefix (UA- or G-) to determine which analytics script template to use.
  • Empty Value Validation: The system correctly prevents saving an empty tracking ID with the error message "Tracking Code Cannot Be Empty".
  • Special Characters Accepted: The system accepts special characters including HTML/script tags without sanitization. This could potentially be a security consideration depending on how the value is rendered in the site analytics script.
  • Track for Administrators Option: The "Track for Administrators" checkbox allows enabling/disabling analytics tracking for admin users.
  • Advanced Settings: The code shows there are Advanced Settings available (domain name, rules) accessible via a separate dialog, but these were not visible in the basic configuration panel tested.
  • Universal Analytics Warning: According to the code review, when a UA- prefix tracking ID is used, a warning message should be displayed about Universal Analytics deprecation, but this was not observed during testing (may require page refresh or be conditionally shown).
